# Design: pricing_data - LLM 定价数据平台

Generated by /office-hours on 2026-03-28
Branch: main
Repo: burncloud/pricing_data
Status: DRAFT
Mode: Builder

## Problem Statement

LLM API 价格分散在各个供应商网站，没有统一的机器可读数据源。burncloud 作为 LLM API 代理服务需要一个可靠、自动更新的定价数据源。同时开源给社区使用。

## What Makes This Cool

- **单一数据源**：132 个模型（当前）的定价数据，一处维护
- **自动更新**：每日 CI 从多个数据源抓取最新价格
- **价格比对**：OpenRouter vs 原厂价格对比，发现差异
- **版本控制**：Git 追踪每次价格变动历史
- **开箱即用**：标准 JSON Schema，方便集成

## Constraints

- 数据必须机器可读（JSON + Schema）
- 每日自动更新，无需人工干预
- 支持国际模型（OpenRouter）和国内模型（百度/阿里/讯飞/智谱/月之暗面/MiniMax）
- 价格需要与原厂比对验证
- burncloud 作为主要消费者

## Premises

1. **OpenRouter API 提供可靠的定价数据** - 需要验证其与原厂价格的一致性
2. **国内模型 API 有公开的定价接口** - 需要确认各供应商的 API 可用性（已在 Next Steps 中标记为第一步）
3. **每日更新频率足够** - 价格不会在同一天内多次变动
4. **Git 历史足以追踪价格变化** - 不需要单独的历史数据库

## Approaches Considered

### Approach A: 最小可行版
只有主 JSON + 简单抓取脚本
- **Effort**: M
- **Risk**: Low
- **Pros**: 简单，快速实现
- **Cons**: 无价格比对，无原始数据保留

### Approach B: 完整版 (已选)
多数据源 + 价格比对 + 完整目录结构
- **Effort**: L
- **Risk**: Low
- **Pros**: 完整追踪、比对、历史记录
- **Cons**: 代码维护量稍大

### Approach C: 纯数据版
只保留 JSON，手动更新
- **Effort**: S
- **Risk**: Low
- **Pros**: 最简单
- **Cons**: 手动维护成本高，不可持续

## Recommended Approach: 完整版

## 错误处理策略

### 网络错误
- **重试策略**：失败后指数退避重试 3 次（1s, 2s, 4s）
- **超时设置**：单个 API 请求超时 30 秒
- **失败隔离**：单个数据源失败不影响其他源的抓取

### 数据源不可用
- **保留旧数据**：sources/*.json 保留上次成功抓取的数据
- **告警阈值**：连续 3 天失败创建 GitHub issue
- **降级策略**：使用旧数据 + 更新 fetched_at 为失败时间

### 网页抓取失败
- **结构变化检测**：抓取失败时检查 HTML 结构是否变化
- **合规要求**：遵守 robots.txt，请求间隔 >= 1 秒

## Schema 演进策略

- **向后兼容**：新增字段不影响现有消费者
- **版本控制**：`schema_version` 字段标识版本
- **变更流程**：
  1. 更新 schema.json
  2. 更新 merge_pricing.py 处理新字段
  3. 更新 README 说明新字段含义

## 模型生命周期管理

- **新增模型**：自动添加到 pricing.json
- **弃用模型**：设置 `deprecated: true` + `sunset_date`
- **删除模型**：保留 30 天后从 pricing.json 移除，sources/ 保留历史

## 决策（已确认）

1. **价格冲突处理**：
   - 优先级：原厂 API > OpenRouter > 手动维护
   - >1% 差异创建 issue
2. **国内模型 API 可用性**：优先 API，次选网页抓取，最后手动维护
3. **更新方式**：直接 push 到 main（数据更新不需要 code review）

### 模型 ID 规范化规则
- sources/ 中保留原始 ID（如 `openai/gpt-4o`）
- pricing.json 中去除供应商前缀（`openai/gpt-4o` → `gpt-4o`）
- 国内模型保持原名（如 `qwen-max`, `glm-4`）

### 货币处理
- 国际模型：USD
- 国内模型：CNY（人民币）
- **不做汇率转换**：汇率实时波动，消费者自行处理

### 回滚机制
- 如果 push 后发现数据错误：`git revert HEAD` 自动回滚
- 保留最近 7 天的 sources/*.json 历史
- 验证失败不 push（merge_pricing.py 必须通过 schema 验证）
